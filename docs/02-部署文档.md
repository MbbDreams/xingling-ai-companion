## 快速部署

### 方式一：一键启动脚本（推荐开发环境）

```bash
# 1. 克隆项目
git clone <repository-url>
cd xingling-ai-companion

# 2. 给脚本添加执行权限
chmod +x scripts/*.sh

# 3. 运行一键启动脚本
./scripts/start-all.sh
```

脚本会自动完成：
- ✅ 环境检查（Python/Flutter）
- ✅ 启动 Docker 基础设施（PostgreSQL + Redis）
- ✅ 创建 Python 虚拟环境并安装依赖
- ✅ 初始化数据库表结构
- ✅ 启动后端服务（http://localhost:8000）
- ✅ 启动前端应用（Chrome 浏览器）

## 手动部署

### 步骤 1：启动基础设施

#### 使用 Docker（推荐）

```bash
# 启动 PostgreSQL 和 Redis
docker-compose up -d

# 查看运行状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

#### 或使用本地服务

确保以下服务已启动：
- PostgreSQL: `localhost:5433`
- Redis: `localhost:6380`

---

### 步骤 2：后端部署

#### 2.1 创建虚拟环境

```bash
cd backend

# 创建虚拟环境
python3 -m venv .venv

# 激活虚拟环境
# Linux/Mac:
source .venv/bin/activate

# Windows:
.venv\Scripts\activate
```

#### 2.2 安装依赖

```bash
pip install -r requirements.txt
```

#### 2.3 配置环境变量

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑 .env 文件
vim .env
```

**环境变量说明：**

```env
# 数据库配置
DATABASE_URL=postgresql+asyncpg://xingling:xingling_dev@localhost:5433/xingling_ai

# Redis 配置
REDIS_URL=redis://localhost:6380/0

# OpenAI API（用于 AI 对话）
OPENAI_API_KEY=your-openai-api-key
OPENAI_MODEL=gpt-4o
OPENAI_BASE_URL=https://api.openai.com/v1  # 可选，用于代理

# JWT 密钥（生产环境必须修改！）
SECRET_KEY=your-secret-key-here-change-in-production
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=30
REFRESH_TOKEN_EXPIRE_DAYS=7

# CORS 配置（开发环境）
CORS_ORIGINS=["http://localhost:8080","http://localhost:3000","http://localhost:5000","http://localhost:60494"]

# 日志级别
LOG_LEVEL=INFO
```

#### 2.4 初始化数据库

```bash
# 完整初始化（创建表 + 插入初始数据）
python init_db.py

# 仅检查表结构
python init_db.py check

# 仅插入初始数据
python init_db.py seed
```

**创建的表结构：**

| 表名 | 说明 |
|------|------|
| `users` | 用户表（含认证、个人信息、货币） |
| `companions` | AI 伴侣表 |
| `conversations` | 对话会话表 |
| `messages` | 消息表 |
| `memories` | 记忆表（含向量嵌入） |
| `diary_entries` | 日记表 |
| `growth_milestones` | 成长里程碑表 |
| `shop_items` | 商店商品表 |
| `user_items` | 用户购买记录表 |
| `analytics_events` | 分析事件表 |

#### 2.5 启动后端服务

```bash
# 开发模式（热重载）
python -m uvicorn app.main:app --reload --port 8000

# 生产模式
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 4
```

服务启动后访问：
- API 文档：http://localhost:8000/docs
- 健康检查：http://localhost:8000/health

## 生产环境部署

### 后端生产部署

#### 使用 Gunicorn + Uvicorn

```bash
# 安装 Gunicorn
pip install gunicorn

# 启动生产服务
gunicorn app.main:app \
  -w 4 \
  -k uvicorn.workers.UvicornWorker \
  --bind 0.0.0.0:8000 \
  --access-logfile - \
  --error-logfile - \
  --log-level info
```

#### 使用 Systemd 服务

创建 `/etc/systemd/system/xingling-backend.service`：

```ini
[Unit]
Description=Xingling AI Companion Backend
After=network.target

[Service]
Type=simple
User=xingling
WorkingDirectory=/opt/xingling-ai-companion/backend
Environment=PATH=/opt/xingling-ai-companion/backend/.venv/bin
Environment=PYTHONPATH=/opt/xingling-ai-companion/backend
ExecStart=/opt/xingling-ai-companion/backend/.venv/bin/gunicorn app.main:app -w 4 -k uvicorn.workers.UvicornWorker --bind 0.0.0.0:8000
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable xingling-backend
sudo systemctl start xingling-backend
sudo systemctl status xingling-backend
```

#### 使用 Nginx 反向代理

```nginx
server {
    listen 80;
    server_name api.xingling.ai;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

## Docker 部署

### 构建后端镜像

```dockerfile
# Dockerfile
FROM python:3.12-slim

WORKDIR /app

# 安装系统依赖
RUN apt-get update && apt-get install -y \
    gcc \
    postgresql-client \
    && rm -rf /var/lib/apt/lists/*

# 复制依赖文件
COPY requirements.txt .

# 安装 Python 依赖
RUN pip install --no-cache-dir -r requirements.txt

# 复制应用代码
COPY app/ ./app/
COPY init_db.py .

# 暴露端口
EXPOSE 8000

# 启动命令
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

构建并运行：

```bash
# 构建镜像
docker build -t xingling-backend:latest .

# 运行容器
docker run -d \
  --name xingling-backend \
  -p 8000:8000 \
  --env-file .env \
  xingling-backend:latest
```

### Docker Compose 完整部署

```yaml
# docker-compose.prod.yml
version: '3.8'

services:
  postgres:
    image: postgres:15-alpine
    environment:
      POSTGRES_USER: xingling
      POSTGRES_PASSWORD: ${DB_PASSWORD}
      POSTGRES_DB: xingling_ai
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "5433:5432"
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U xingling"]
      interval: 5s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    volumes:
      - redis_data:/data
    ports:
      - "6380:6379"
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 5s
      timeout: 3s
      retries: 5

  backend:
    build: ./backend
    environment:
      DATABASE_URL: postgresql+asyncpg://xingling:${DB_PASSWORD}@postgres:5432/xingling_ai
      REDIS_URL: redis://redis:6379/0
      SECRET_KEY: ${SECRET_KEY}
      OPENAI_API_KEY: ${OPENAI_API_KEY}
    ports:
      - "8000:8000"
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_healthy
    restart: unless-stopped

  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf
      - ./flutter_app/build/web:/usr/share/nginx/html
    depends_on:
      - backend
    restart: unless-stopped

volumes:
  postgres_data:
  redis_data:
```

部署：

```bash
# 启动所有服务
docker-compose -f docker-compose.prod.yml up -d

# 查看日志
docker-compose -f docker-compose.prod.yml logs -f

# 停止服务
docker-compose -f docker-compose.prod.yml down
```

## 常见问题

### Q1: 前端无法连接后端？

**检查清单：**

1. 后端是否运行在正确的地址和端口
   ```bash
   curl http://localhost:8000/health
   ```

2. CORS 配置是否正确
   - 检查 `backend/app/core/config.py` 中的 `cors_origins`
   - 确保包含前端运行的端口

3. 浏览器控制台是否有 CORS 错误
   - 查看 Network 标签页的请求详情

4. 防火墙是否拦截了 8000 端口
   ```bash
   # Linux
   sudo ufw allow 8000
   
   # 或关闭防火墙（仅开发环境）
   sudo ufw disable
   ```

### Q2: 验证码发送失败？

当前使用模拟短信服务，验证码会打印在后端控制台。查看后端日志获取验证码：

```bash
# 如果使用脚本启动
# 查看后端终端窗口的输出

# 如果使用 Docker
docker-compose logs -f backend
```

### Q3: 数据库连接失败？

**检查清单：**

1. PostgreSQL 是否运行
   ```bash
   docker-compose ps
   # 或
   pg_isready -h localhost -p 5433
   ```

2. 数据库是否存在
   ```bash
   docker-compose exec postgres psql -U xingling -l
   ```

3. 连接字符串是否正确
   ```bash
   # 测试连接
   docker-compose exec backend python -c "
   import asyncio
   from sqlalchemy.ext.asyncio import create_async_engine
   engine = create_async_engine('postgresql+asyncpg://xingling:xingling_dev@postgres:5432/xingling_ai')
   async def test():
       async with engine.connect() as conn:
           result = await conn.execute('SELECT 1')
           print(result.scalar())
   asyncio.run(test())
   "
   ```

4. 是否已运行 `init_db.py` 创建表
   ```bash
   cd backend
   python init_db.py
   ```

### Q4: 表结构缺失或字段不全？

运行完整初始化脚本：

```bash
cd backend
source .venv/bin/activate

# 完整重建（会删除现有数据！）
python init_db.py
# 输入 yes 确认

# 或仅检查
python init_db.py check
```

### Q5: Flutter 依赖安装失败？

```bash
cd flutter_app

# 清理缓存
flutter clean

# 修复缓存
flutter pub cache repair

# 重新获取依赖
flutter pub get

# 如果仍失败，尝试升级 Flutter
flutter upgrade
```

### Q6: 脚本没有执行权限？

```bash
chmod +x scripts/*.sh

# 或
chmod +x scripts/start-all.sh
chmod +x scripts/start-backend.sh
chmod +x scripts/start-frontend.sh
```

### Q7: OpenAI API 调用失败？

1. 检查 API Key 是否正确
   ```bash
   # 测试 API Key
   curl https://api.openai.com/v1/models \
     -H "Authorization: Bearer your-api-key"
   ```

2. 检查网络连接（可能需要代理）
   ```env
   # .env
   OPENAI_BASE_URL=https://your-proxy-url/v1
   ```

3. 检查账户余额

### Q8: 内存不足导致服务崩溃？

**优化建议：**

1. 减少 Gunicorn worker 数量
   ```bash
   gunicorn app.main:app -w 2  # 改为 2 个 worker
   ```

2. 限制 PostgreSQL 内存使用
   ```yaml
   # docker-compose.yml
   postgres:
     command: >
       postgres
       -c shared_buffers=256MB
       -c effective_cache_size=768MB
   ```

3. 使用交换分区（Swap）
   ```bash
   sudo fallocate -l 2G /swapfile
   sudo chmod 600 /swapfile
   sudo mkswap /swapfile
   sudo swapon /swapfile
   ```

## 安全建议

1. **修改默认密码**
   - 数据库密码
   - JWT Secret Key
   - Redis 密码（生产环境）

2. **启用 HTTPS**
   - 使用 Let's Encrypt 免费证书
   - 配置 Nginx SSL

3. **限制 CORS**
   - 生产环境只允许特定域名

4. **定期更新依赖**
   ```bash
   pip list --outdated
   pip install -U package-name
   ```

5. **启用防火墙**
   - 只开放必要端口（80, 443, 8000）
