## 1. 用户认证模块

### 功能列表

| 功能 | 状态 | 说明 |

## 3. 记忆系统模块

### 状态：✅ 已实现（向量检索 + 两阶段排序）

### 功能概述

记忆系统让 AI 能够记住用户分享的重要信息，如喜好、经历、重要日期等，并在后续对话中自然引用。采用 **两阶段检索** 策略，结合向量相似度和多维度排序。

### 功能列表

| 功能 | 状态 | 说明 |
|------|------|------|
| 记忆提取 | ✅ | AI 自动从对话中提取重要信息 |
| 向量存储 | ✅ | 使用 pgvector 存储嵌入向量 |
| 两阶段检索 | ✅ | 粗检索 + 精排序 |
| 记忆引用 | ✅ | 对话中自然引用相关记忆 |
| 记忆管理 | ✅ | 查看、添加、删除记忆 |
| 记忆分类 | ✅ | 支持分类标签 |
| 意图驱动检索 | ✅ | 根据意图类型优化检索 |

### 两阶段检索流程

```
┌─────────────────────────────────────────────────────────────┐
│                     两阶段记忆检索                            │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  阶段 1: 粗检索 (Coarse Retrieval)                           │
│  ┌─────────────────────────────────────────────────────┐    │
│  │ - 向量相似度计算 (cosine similarity)                 │    │
│  │ - 返回 Top-K (默认 50)                               │    │
│  │ - 使用 HNSW 索引加速                                 │    │
│  └─────────────────────────────────────────────────────┘    │
│                          │                                   │
│                          ▼                                   │
│  阶段 2: 精排序 (Fine Ranking)                               │
│  ┌─────────────────────────────────────────────────────┐    │
│  │ - 多维度评分：相关性 + 时效性 + 重要度               │    │
│  │ - 时间衰减函数：e^(-λt)                              │    │
│  │ - 返回 Top-N (默认 10)                               │    │
│  └─────────────────────────────────────────────────────┘    │
│                          │                                   │
│                          ▼                                   │
│  意图过滤 (Intent Filtering)                                 │
│  ┌─────────────────────────────────────────────────────┐    │
│  │ - 根据意图类型筛选记忆                               │    │
│  │ - 情感类意图优先情感记忆                             │    │
│  │ - 知识类意图优先事实记忆                             │    │
│  └─────────────────────────────────────────────────────┘    │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 评分公式

```python
# 综合评分 = 相关性 × 重要度系数 × 时效性系数
final_score = relevance_score * importance_factor * recency_factor

# 重要度系数 (1.0 ~ 2.0)
importance_factor = 1.0 + (importance - 3) * 0.25

# 时效性系数 (指数衰减)
recency_factor = exp(-λ * hours_since_created)
```

## 技术债务与优化

### 已知问题

1. **任务系统使用内存存储**
   - 当前 `_completed_tasks` 存储在内存中
   - 重启服务后数据丢失
   - **建议**：使用 Redis 或数据库持久化

2. **商店支付未接入**
   - 仅实现了商品展示
   - 支付流程待接入
   - **建议**：接入微信支付/支付宝

3. **头像上传使用 Base64**
   - 数据库存储大字段
   - **建议**：使用对象存储（OSS/S3）

4. **缺少限流机制**
   - API 没有频率限制
   - **建议**：添加 Redis 限流

### 已完成的优化 ✅

1. **AI Agent 架构升级**
   - 使用 LangGraph 状态机
   - 实现流式输出
   - 首 token < 100ms

2. **多级缓存架构**
   - L1: 内存缓存
   - L2: Redis 缓存
   - L3: PostgreSQL
   - 缓存命中率 ~80%

3. **记忆检索优化**
   - 两阶段检索策略
   - HNSW 向量索引
   - 意图驱动过滤

4. **Prompt 工程优化**
   - 专业人设设计
   - 关系阶段感知
   - 情感响应策略

### 性能优化建议

1. **数据库优化**
   - 添加常用查询索引
   - 消息表按时间分区

2. **缓存优化**
   - Redis 缓存用户信息
   - 缓存商品列表

3. **前端优化**
   - 图片懒加载
   - 消息列表虚拟滚动
