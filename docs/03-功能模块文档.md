## 1. 用户认证模块

### 状态：✅ 已实现

### 功能概述

用户认证模块提供完整的身份验证和授权功能，支持手机号+验证码登录注册，JWT Token 认证机制。

### 功能列表

| 功能 | 状态 | 说明 |
|------|------|------|
| 发送验证码 | ✅ | 模拟短信服务，验证码打印在控制台 |
| 手机号注册 | ✅ | 验证码验证后自动创建用户 |
| 手机号登录 | ✅ | 支持自动注册（新手机号） |
| JWT Token | ✅ | Access Token + Refresh Token 双令牌机制 |
| Token 刷新 | ✅ | 使用 Refresh Token 获取新令牌 |
| 个人资料管理 | ✅ | 查看和修改用户信息 |
| 密码修改 | ✅ | 原密码验证后修改 |
| 退出登录 | ✅ | 清除本地 Token |

### API 接口

```
POST /api/v1/auth/send-code          # 发送验证码
POST /api/v1/auth/register           # 注册
POST /api/v1/auth/login              # 登录
POST /api/v1/auth/refresh            # 刷新 Token
GET  /api/v1/auth/me                 # 获取当前用户
PUT  /api/v1/auth/me                 # 更新个人资料
POST /api/v1/auth/change-password    # 修改密码
POST /api/v1/auth/logout             # 退出登录
```

### 数据模型

**User 表**
```python
class User:
    id: int                    # 主键
    phone: str                 # 手机号（唯一）
    nickname: str              # 昵称
    avatar: str                # 头像 URL
    email: str                 # 邮箱
    gender: str                # 性别 (male/female/other)
    birthday: date             # 生日
    bio: str                   # 个人简介
    location: str              # 所在地
    website: str               # 个人网站
    coins: int                 # 星币数量
    is_vip: bool               # VIP 状态
    vip_expire_at: datetime    # VIP 过期时间
    hashed_password: str       # 密码哈希
    auth_provider: str         # 认证方式 (phone)
    created_at: datetime       # 创建时间
    updated_at: datetime       # 更新时间
    last_login_at: datetime    # 最后登录时间
```

### 前端页面

- `LoginPage` - 登录页面
- `RegisterPage` - 注册页面

### 技术实现

**后端**
- `backend/app/api/routes/auth.py` - API 路由
- `backend/app/services/auth_service.py` - 认证服务
- JWT 使用 `python-jose` 库
- 密码使用 `passlib` 进行 bcrypt 哈希

**前端**
- `flutter_app/lib/screens/auth/login_page.dart`
- `flutter_app/lib/screens/auth/register_page.dart`
- `flutter_app/lib/services/auth_service.dart`
- Token 存储使用 `flutter_secure_storage`

## 2. 智能对话模块

### 状态：✅ 已实现

智能对话模块是应用的核心功能，用户可以与 AI 伴侣"晚星"进行自然语言对话。基于 OpenAI GPT-4 实现，支持情感理解和记忆引用。

### 功能列表

POST /api/v1/chat/send               # 发送消息

## 3. 记忆系统模块

### 状态：✅ 已实现

### 功能概述

记忆系统让 AI 能够记住用户分享的重要信息，如喜好、经历、重要日期等，并在后续对话中自然引用，创造更个性化的陪伴体验。

### 功能列表

| 功能 | 状态 | 说明 |
|------|------|------|
| 记忆提取 | ✅ | AI 自动从对话中提取重要信息 |
| 记忆存储 | ✅ | 使用 pgvector 存储向量嵌入 |
| 记忆检索 | ✅ | 基于相似度的记忆检索 |
| 记忆引用 | ✅ | 对话中自然引用相关记忆 |
| 记忆管理 | ✅ | 查看、添加、删除记忆 |
| 记忆分类 | ✅ | 支持分类标签 |

### API 接口

```
GET  /api/v1/memory/list             # 获取记忆列表
POST /api/v1/memory                  # 添加记忆
DELETE /api/v1/memory/{id}           # 删除记忆
```

### 数据模型

**Memory 表**
```python
class Memory:
    id: int                    # 主键
    user_id: int               # 用户ID
    content: str               # 记忆内容
    category: str              # 分类标签
    embedding: vector          # 向量嵌入 (pgvector)
    importance: int            # 重要程度 (1-5)
    created_at: datetime       # 创建时间
    updated_at: datetime       # 更新时间
```

### 前端页面

- `MemoryScreen` - 记忆管理页面

### 技术实现

**后端**
- `backend/app/api/routes/memories.py`
- `backend/app/services/memory_service.py`
- 使用 `pgvector` 扩展存储向量
- OpenAI Embedding API 生成向量
- 余弦相似度检索相关记忆

**前端**
- `flutter_app/lib/screens/memory/memory_screen.dart`
- 记忆卡片列表展示
- 支持添加和删除记忆

### 记忆提取逻辑

```python
# 从对话中提取记忆的 Prompt
"""
从以下对话中提取用户的重要信息（喜好、经历、重要日期等）。
只提取事实性信息，不要推测。
格式：JSON 数组，每个记忆包含 content 和 category
"""
```

## 技术债务与优化

### 已知问题

1. **任务系统使用内存存储**
   - 当前 `_completed_tasks` 存储在内存中
   - 重启服务后数据丢失
   - **建议**：使用 Redis 或数据库持久化

2. **商店支付未接入**
   - 仅实现了商品展示
   - 支付流程待接入
   - **建议**：接入微信支付/支付宝

3. **头像上传使用 Base64**
   - 数据库存储大字段
   - **建议**：使用对象存储（OSS/S3）

4. **缺少限流机制**
   - API 没有频率限制
   - **建议**：添加 Redis 限流

### 性能优化建议

1. **数据库优化**
   - 添加常用查询索引
   - 消息表按时间分区

2. **缓存优化**
   - Redis 缓存用户信息
   - 缓存商品列表

3. **前端优化**
   - 图片懒加载
   - 消息列表虚拟滚动
