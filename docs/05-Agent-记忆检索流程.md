# AI Agent 记忆检索流程设计

## 核心问题

**用户提问**: 用户聊天时，Agent 首先会去检索向量数据库吗？

**答案**: **不是直接检索**，而是先**理解用户意图**，再**智能决定检索策略**。

---

## 关键设计决策

### 1. 为什么先理解再检索？

**问题**: 如果直接检索，可能会：
- 检索到不相关的记忆
- 在紧急情况下浪费时间
- 无法根据情绪调整检索策略

**解决方案**: 
- 先理解意图和情绪
- 根据意图构建不同的检索查询
- 根据情绪过滤记忆

### 2. 为什么需要两阶段检索？

**阶段 1 (粗排)**: 
- 快速获取候选集 (15条)
- 基于向量相似度
- 扩大召回范围

**阶段 2 (精排)**:
- 多维度精细评分
- 考虑情绪、时间、重要性
- 选择最相关的记忆

**好处**: 平衡效率和准确性

### 3. 为什么意图决定记忆数量？

| 场景 | 记忆数量 | 原因 |
|------|----------|------|
| 用户难过求安慰 | 4条 | 需要引用很多过往经历来表达"我记得"、"我理解" |
| 用户提问 | 1条 | 聚焦回答，过多记忆会分散注意力 |
| 用户需要空间 | 0条 | 不要引用，给予空间 |

### 4. 实时记忆提取的价值

**传统方式**: 定期批量处理
**我们的方式**: 每次对话实时提取

**好处**:
- 即时学习用户新信息
- 下次对话就能引用
- 更自然的长期记忆效果

## 流程对比

### 传统 RAG 流程
```
用户消息 → 向量检索 → 生成回复
```
**问题**: 无脑检索，不考虑上下文

### 我们的优化流程
```
用户消息 → 理解(意图+情绪) → 智能检索(意图驱动) → 精排 → 选择 → 生成
```
**优势**: 
- 意图驱动检索策略
- 情绪感知记忆过滤
- 动态调整记忆数量
- 实时学习新记忆

```python

agent = CompanionAgentV2(db, openai_api_key)

    user_id=1,
    user_name="小明",

# 返回包含:
# - response: AI回复
# - emotion: 检测到的情绪
# - intent: 识别的意图
# - memories_used: 使用的记忆列表
# - strategy: 响应策略
# - processing_time_ms: 处理时间
```

## 总结

最合理的记忆检索流程:

1. **先理解，后检索** - 意图和情绪指导检索策略
2. **两阶段检索** - 粗排召回 + 精排排序
3. **意图驱动数量** - 不同场景使用不同数量的记忆
4. **实时学习** - 每次对话都在提取和存储新记忆
5. **策略化响应** - 根据意图选择不同的回复策略

这让 AI 回复更智能、更自然、更像真人！
